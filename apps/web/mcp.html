How it works

MCP = a data channel for your AI assistant

MCP (Model Context Protocol) is an open standard that lets AI assistants use external tools and data sources. With TxReceipts MCP, your assistant reads wallet data directly from the Base blockchain.

MCP Tools

Read-only accounting tools

The TxReceipts MCP server exposes 6 tools. All are read-only — no signing, no transfers, no approvals, no writes.

Tool 1

list_networks

Lists supported networks (Base, Ethereum, Optimism, Arbitrum, Polygon). Returns each network's RPC URL, explorer address, and capabilities.

Tool 2

get_wallet_activity

Fetches a wallet's transaction history and token transfers. Accepts network, wallet address, and limit parameters.

Tool 3

get_transaction_summary

Returns a detailed summary of a single transaction: value sent, gas fees, token movements, block time, and status.

Tool 4

get_public_receipt

Fetches a public receipt by a valid TXR-XXXX-XXXX receipt ID.

Tool 5

create_receipt

Creates an accounting receipt using an API key. Requires authentication — only authorized projects can use this tool.

Tool 6

get_pre_accounting_report

Generates a comprehensive pre-accounting report for a wallet: token flows, gas spend, incoming/outgoing summary, and network capabilities. Runs with full capability support on Base.

Setup

Add to your MCP client

VS Code (settings.json) 
{
  "mcp": {
    "servers": {
      "txreceipts": {
        "type": "url",
        "url": "https://txreceipts-api.evpc77.workers.dev/mcp"
      }
    }
  }
}

Add this to your VS Code settings. Copilot Chat will automatically discover and use TxReceipts MCP tools.

Claude Desktop (claude_desktop_config.json) 
{
  "mcpServers": {
    "txreceipts": {
      "type": "url",
      "url": "https://txreceipts-api.evpc77.workers.dev/mcp"
    }
  }
}

Add the config and restart Claude. You can start asking Base wallet questions immediately.

Security

Read-only. No approvals. No transfers.

  • TxReceipts MCP is read-only — it cannot sign, approve, transfer, or swap
  • No private keys, seed phrases, or API tokens are sent through the MCP context
  • Wallet balances and transactions are fetched from public blockchain data only
  • The create_receipt tool requires an additional API key and is optional
  • Rate limits are enforced per MCP request
  • The AI assistant cannot call MCP tools without your permission

FAQ

Frequently asked questions

Which AI assistants work with MCP?

Claude Desktop, VS Code Copilot Chat, Cursor, GitHub Copilot, and any client that supports the MCP protocol.

Is there a fee to use MCP?

No. MCP access is completely free and does not require an API key. Just add the server URL to your MCP client.

Can my assistant transfer tokens from my wallet?

No. All MCP tools are read-only. Your assistant can read blockchain data but cannot sign, approve, or transfer anything.

Which networks are supported?

Base (default), Ethereum, Optimism, Arbitrum, and Polygon. Base runs with full capability support (MCP, X402, Paymaster, Base Account, Builder Codes).

Is the pre-accounting report an official accounting record?

No. The report is a pre-accounting record and is not an official invoice, tax filing, or e-ledger entry. Consult a licensed accountant for official bookkeeping.
