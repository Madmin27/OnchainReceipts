Product

Not another explorer view.

User receipt inbox

Users connect a wallet, browse network transactions by date, generate receipts, and download PNG files for monthly reconciliation.

Dapp intent metadata

Dapps can submit the business context behind a transaction: expected assets, fees, merchant references, categories, and receipt copy.

Onchain verification

The engine checks transaction success, token transfers, gas paid, observed fees, and whether the app's claim matched the Base transaction.

Security

Read-only by default. Explicit when shared.

TxReceipts should never ask for approvals, spending permissions, private keys, or seed phrases. Wallet signatures are used only for login and quota ownership.

  • No token approvals required
  • Untrusted dapp metadata is verified before display
  • Escaped receipt rendering, no arbitrary HTML
  • Wallet, IP, and API-key rate limits
  • Duplicate transaction dedupe for fair billing
